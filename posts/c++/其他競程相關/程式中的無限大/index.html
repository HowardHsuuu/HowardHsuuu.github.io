程式中的無限大$_\infty$#

在程式競賽中，遇到如「比較出最小值」之類的需求時，往往需要一個「無限大」的初始值，但在程式中要如何表現「無限大」呢？多大才能算「無限大」呢？以下三種想法提供參考：

1.根據題目需求而定#

從題目給的資料大小範圍、使用的演算法等條件預估出這隻程式中將出現的最大數值大概是多少，那麼只要確保「無限大」的設置恆大於這個預估可能出現的最大數值即可。 優點大概是不用多背東西，缺點則是每次都要花時間思考又容易因為粗心或不夠了解演算法而設置不夠大的「無限大」，導致執行過程出現BUG。

2.使用資料型態的上限#

直接將「無限大」設置為該資料型態的上限，如 int 就可以設置為0x7fffffff，即為2147483647，這樣一來也確保了不會出現比「無限大」還大的其他 int 型態變數。 優點大概是比較無腦，也能完全確保不出現大於「無限大」的數，缺點則是這樣的設置方法無法滿足$\infty + \infty=\infty$ 的性質，由於溢位的關係，此種設置下 $\infty + \infty=-2$。

3.INF = 0x3f3f3f3f#

這是在欣賞某位電神的程式時發現的設置法，自己稍微查了一下資料發現真的是很不錯的選擇。 0x3f3f3f3f的十進制是1061109567，和 int 的上限是同一個數量級，因此大部分情況下不太會出現比他大的值，也就可以當作「無限大」使用。 此種設置下 $\infty + \infty=2122219134$，尚未產生溢位，相當於滿足 $\infty + \infty=\infty$ 的性質。

這種設置還有一個額外的好處，就是可以直接使用 memset() 初始化陣列，而不用自己寫迴圈， memset() 是按記憶體的byte執行的，因此通常只有陣列初始化為0或-1時能使用 (0的每個byte皆為0，-1亦然)，而0x3f3f3f3f的每個byte都是0x3f，因此想將陣列初始化成「無限大」時只要

memset(arr, 0x3f, sizeof(arr))

綜上所述，只要不是出現比1061109567大的數值的特殊情況，正常情況下將「無限大」設為0x3f3f3f3f是不錯的選擇。
