Hello World - C++的基本架構

程式基本架構 #include <iostream> using namespace std; int main(){ //code here return 0; } 1.C++中的語句結尾須加上**;。 2.以#**開頭為程式的預處理，也就是程式編譯前執行的指令。 3.第一行為引入標頭檔，可以想像成給電腦一本參考的字典，以便認識你所使用的函式等。 iostream是IO(輸入輸出)與stream(流)，也就是與輸入輸出有關的函式庫。 4.第二行為使用標準命名空間，想像一下如果班上有兩個小明，那老師在點名時是不是就不知道是在叫誰？命名空間可以視為一個確定不會出現相同人名的班級，也就是一個名詞指對應到一個意義。 若不加此行，在用到此命名空間的成員時須加上std::，如std::cout。 5.main()為主函式，主函式程式放在{}中，依上到下循序執行。 6.以//開頭者為單行註解，註解是寫給人看的，電腦會跳過，另有多行註解，以/*與*/包裹。 7.return 0 使程式結束。 輸出 - 第一支程式 Hello World ! #include <iostream> using namespace std; int main(){ cout<<"Hello World !"<<endl; return 0; }

程式中的無限大∞

程式競賽中，遇到如「比較出最小值」之類的需求時，往往需要一個「無限大」的初始值，但在程式中要如何表現「無限大」呢？多大才算「無限大」呢？以下三種想法提供參考： 1.根據題目需求而定 從題目給的資料大小範圍、使用的演算法等條件預估出這隻程式中將出現的最大數值大概是多少，那麼只要確保「無限大」的設置恆大於這個預估可能出現的最大數值即可。 優點大概是不用多背東西，缺點則是每次都要花時間思考又容易因為粗心或不夠了解演算法而設置不夠大的「無限大」，導致執行過程出現BUG。 2.使用資料型態的上限 直接將「無限大」設置為該資料型態的上限，如 int 就可以設置為0x7fffffff，即為2147483647，這樣一來也確保了不會出現比「無限大」還大的其他 int 型態變數。 優點大概是比較無腦，也能完全確保不出現大於「無限大」的數，缺點則是這樣的設置方法無法滿足$\infty + \infty=\infty$ 的性質，由於溢位的關係，此種設置下 $\infty + \infty=-2$。 3.INF = 0x3f3f3f3f 這是在欣賞某位電神的程式時發現的設置法，自己稍微查了一下資料發現真的是很不錯的選擇。 0x3f3f3f3f的十進制是1061109567，和 int 的上限是同一個數量級，因此大部分情況下不太會出現比他大的值，也就可以當作「無限大」使用。 此種設置下 $\infty + \infty=2122219134$，尚未產生溢位，相當於滿足 $\infty + \infty=\infty$ 的性質。 這種設置還有一個額外的好處，就是可以直接使用 memset() 初始化陣列，而不用自己寫迴圈， memset() 是按記憶體的byte執行的，因此通常只有陣列初始化為0或-1時能使用 (0的每個byte皆為0，-1亦然)，而0x3f3f3f3f的每個byte都是0x3f，因此想將陣列初始化成「無限大」時只要： memset(arr, 0x3f, sizeof(arr)) 綜上所述，只要不是出現比1061109567大的數值的特殊情況，正常情況下將「無限大」設為0x3f3f3f3f是不錯的選擇。

Disjoint Set Union (DSU)

互斥集為彼此沒有交集的集合，也可以說每個元素只分別屬於某一個集合。 Disjoint Set Union 簡稱 DSU，中文為並查集，是一種以 union-find algorithm 處理查詢(find)與合併(Union)互斥集的資料結構。 簡陋的DSU功能示意動畫： 概念 DSU的資料結構是一個有向圖，每個節點皆指向自己的「老大」節點，當某個節點指向自己(即自己為自己的老大)時就代表其為此集合的最上層，我們便以此節點代表這個集合。具有相同最上層的所有節點屬於同一集合。 或者，可以把每個集合想像為一棵樹，每個節點都記錄自己的父節點，當某節點紀錄的是自己則代表其為此樹的根節點，而我們以根節點代表每棵不同的樹(集合)。 實作 我們可以藉由陣列來實現DSU： 以index為節點編號，每個index儲存自己的「老大」，初始時每個index的老大都設為自己。 可以用另一陣列儲存集合大小，且只有當index為最上層(根節點)時，所儲存的大小才有意義。 查詢節點所處集合 — find() 要找到某節點所屬的集合，需要不斷向上溯源，找到老大的老大再找到老大的老大，遞迴直到找到老大是自己的節點(最上層)，也就是能代表此集合的節點，參考程式： int find(int i){ if(boss[i] == i) return i; return find(boss[i]); } 想想看，若所有節點排成一排，每個節點都指向自己前面的節點，但真正的老大(最上層)位於最前方，則若要查詢第一個節點所屬的集合，要全部節點跑過一遍才能找到最上層，也就是此算法在最糟的情況下需要 $O(n)$ 才能找到最上層。 為了避免每次查詢都要浪費時間，我們可以在找到「最上層」時通知查詢時路過的每個節點真正的老大(最上層)是誰，並讓他們直接指向它。此優化稱為路徑壓縮，參考程式(修改第三行)： return boss[i] = find(boss[i]); //路徑壓縮 自製的簡陋示意動畫： 合併兩節點所在集合 — Union() 要使兩個集合合併(以A集合融入B集合為例)，只要讓A集合的「最上層」指向B集合的「最上層」並更新B集合的大小即可。參考程式： void Union(int a, int b){ int boss_a = find(a), boss_b = find(b); //先找a與b的老大 size[boss_b] += size[boss_a]; boss[boss_a] = boss_b; } 想想看，如果要在現實生活中操作合併集合的動作，是不是會選擇在每次合併時都讓較小的集合融進較大的集合？ 這種直覺性或根據經驗的較佳做法被稱為Heuristic(啟發式)，因此以下改良被稱為啟發式合併(以下程式為union by size，另有union by rank，在此不贅述)，參考程式(在第三行加入)：...

IO 優化 (Fast IO)

在程式競賽中，偶爾會看到輸入輸出量較大的題目建議參賽者使用 scanf() 和 printf() 以避免TLE，或者使用 cin 和 cout 時加入： ios_base::sync_with_stdio(false); cin.tie(0); 在操作上 cin 和 cout 比 scanf() 和 printf() 更方便直觀，且保證型別安全。效率上cin 和 cout 卻似乎輸了不少，這是為什麼呢？ 兩個輸入輸出系統 (stdio & iostream) C++中有兩種IO系統，stdio 源自於C語言，而 iostream 來自C++，為了避免在混用時發生問題，C++會自動同步兩者，然而這樣的操作便會降低效率。 所以如果確定不會有混用的需求，可以透過指令解除同步： (用了之後要記得 stdio 和 iostream 不能混用) ios_base::sync_with_stdio(false); 緩衝區 (buffer) 在資訊被輸出前，會先到「緩衝區」等待輸出，而分次輸出很耗資源，因此釋放緩衝區的次數是影響效率的一大變因。 在 iostream 中，可以用 cout<<flush; 來釋放緩衝區， cin 則因為使用上的考量(讓使用者在輸入時能看見先前操作的輸出結果)，具有自動釋放緩衝區的功能。 cin 的效率之所以比 scanf() 差便是因為 scanf() 並不會自動釋放緩衝區，我們可以透過指令來解除 cin 的自動釋放： cin.tie(0); 另外，使用 endl 換行也會釋放緩衝區，即 endl 等於 flush + '\n'。所以如果不是互動題，建議換行使用 '\n' 就可以了。
