Disjoint Set Union (DSU)

互斥集為彼此沒有交集的集合，也可以說每個元素只分別屬於某個集合。 Disjoint Set Union 簡稱 DSU，中文為並查集，是一種以 union-find algorithm 處理查詢(find)與合併(Union)互斥集的資料結構。 簡陋的DSU功能示意動畫： 原理 DSU的資料結構是一個有向圖，每個節點皆指向自己的「老大」節點，當某個節點指向自己(即自己為自己的老大)時就代表其為此集合的最上層，我們便以此節點代表這個集合。具有相同最上層的所有節點屬於同一集合。 或者，也可以把每個集合想像為一棵樹，每個節點都記錄自己的父節點，當某節點紀錄的是自己則其為此樹的根節點，而我們以根節點代表每棵不同的樹(集合)。 實作 我們可以藉由陣列來實現DSU： 以index為節點編號，每個index儲存自己的「老大」，初始時每個index的老大都設為自己。 用另一陣列儲存集合大小，且只有當index為最上層(根節點)時，所儲存的大小才有意義。 查詢節點所處集合 — find() 要找到某節點所處的集合，需要不斷向上溯源，找到老大的老大再找到老大的老大，遞迴直到找到老大是自己的節點(最上層)，也就是能代表此集合的節點，參考程式： int find(int i){ if(boss[i] == i) return i; return find(boss[i]); } 想想看，若所有節點排成一排，每個節點都指向自己前面的節點，但真正的老大(最上層)位於最前方，則若要查詢第一個節點所屬的集合，要全部節點跑過一遍才能找到最上層，也就是此算法在最糟的情況下需要 $O(n)$ 才能找到最上層。 為了避免每次查詢都要浪費時間，我們可以在找到「最上層」時通知查詢時路過的每個節點真正的老大(最上層)是誰，並讓他們直接指向它。此優化稱為路徑壓縮，參考程式(修改第三行)： return boss[i] = find(boss[i]); //路徑壓縮 自製的簡陋示意動畫： 合併兩節點所在集合 — Union() 要使兩個集合合併(以A集合融入B集合為例)，只要讓A集合的「最上層」指向B集合的「最上層」並更新B集合的大小即可。參考程式： void Union(int a, int b){ int boss_a = find(a), boss_b = find(b); //先找a與b的老大 size[boss_b] += size[boss_a]; boss[boss_a] = boss_b; } 想想看，如果要在現實生活中操作合併集合的動作，是不是會選擇在每次合併時都讓較小的集合融進較大的集合？ 這種直覺性或根據經驗的較佳做法被稱為Heuristic(啟發式)，因此以下改良為啟發式合併(union by size，另有union by rank在此不贅述)，參考程式(在第七行加入)：...

IO 優化 (Fast IO)

在程式競賽中，偶爾會看到輸入輸出量較大的題目建議參賽者使用 scanf() 和 printf() 以避免TLE，或者使用 cin 和 cout 時加入： ios_base::sync_with_stdio(false); cin.tie(0); 在操作上 cin 和 cout 比 scanf() 和 printf() 更方便直觀，且保證型別安全。效率上cin 和 cout 卻似乎輸了不少，這是為什麼呢？ 兩個輸入輸出系統 (stdio & iostream) C++中有兩種IO系統，stdio 源自於C語言，而 iostream 來自C++，為了避免在混用時發生問題，C++會自動同步兩者，然而這樣的操作便會降低效率。 所以如果確定不會有混用的需求，可以透過指令解除同步： (用了之後要記得 stdio 和 iostream 不能混用) ios_base::sync_with_stdio(false); 緩衝區 (buffer) 在資訊被輸出前，會先到「緩衝區」等待輸出，而分次輸出很耗資源，因此釋放緩衝區的次數是影響效率的一大變因。 在 iostream 中，可以用 cout<<flush; 來釋放緩衝區， cin 則因為使用上的考量(讓使用者在輸入時能看見先前操作的輸出結果)，具有自動釋放緩衝區的功能。 cin 的效率之所以比 scanf() 差便是因為 scanf() 並不會自動釋放緩衝區，我們可以透過指令來解除 cin 的自動釋放： cin.tie(0); 另外，使用 endl 換行也會釋放緩衝區，即 endl 等於 flush + '\n'。所以如果不是互動題，建議換行使用 '\n' 就可以了。
